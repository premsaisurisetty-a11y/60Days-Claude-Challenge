### Improvements Made

- Converted resume from a two-column layout to a single-column ATS-friendly format.
- Removed icons, images, and decorative elements that ATS systems cannot parse effectively.
- Added a Professional Summary section to improve keyword density.
- Enhanced project descriptions using action verbs and measurable impact.
- Added relevant technical concepts such as:
  - Data Structures
  - Algorithms
  - Full-Stack Development
  - Design Thinking
- Improved overall formatting consistency and readability.

---

## Key Learnings

### 1. ATS-Friendly Design Matters

Many companies use Applicant Tracking Systems (ATS) to filter resumes before they reach recruiters. A clean single-column format significantly improves parsing accuracy.

### 2. Keywords Improve Visibility

Including relevant technical keywords such as Java, Python, SQL, Data Structures, and Full-Stack Development increases ATS matching potential.

### 3. Strong Project Descriptions Are Important

Projects should explain:
- What was built
- Technologies used
- Problems solved
- Impact created

### 4. Professional Summary Adds Value

A concise summary helps recruiters quickly understand a candidate's strengths and career goals.

### 5. AI Can Improve Career Documents

AI can identify weaknesses, optimize wording, improve formatting, and make resumes more recruiter-friendly.
